# 需求文档说明：字段标准置顶 — 设计说明

**日期**：2026-05-15  
**状态**：已定稿（粒度 A + 五列表格）  
**范围**：`需求文档说明`页面（`ModuleDocsPage`）信息结构与字段文档写法；不含具体业务字段填全清单。

---

## 1. 目标

在「需求文档说明」中，为每个**页面/路由粒度**的模块条目增加**置顶的字段标准**，与现有自由文案说明并存：读者先掌握字段口径（校验、来源、演示策略），再阅读列表行为、交互与演示说明。

---

## 2. 粒度与信息架构

- **粒度**：选 **A — 按页面/路由**。每个导航上的「页面」在文档中对应一个 `item`（如「实例管理」），**不**为「新增实例抽屉」「编辑实例抽屉」单独建顶层条目。
- **单条目内结构（自上而下）**：
  1. **字段标准**（置顶；有表单或关键筛选则填写；若该页无独立字段口径，写一句「本页无独立字段口径」即可）
  2. **说明**（沿用现有 bullets：列表、提示、权限、演示等非逐字段内容）

推荐采用 **方案 1**：置顶字段表 + 下方保留原 bullets；后续若重复多，可再收紧为「字段只进表、说明只写非字段」。

---

## 3. 字段标准表 — 五列定义

每行描述一个界面字段（或条件出现的字段）；列名固定，便于检索与维护。

| 列名 | 含义 | 约定 |
|------|------|------|
| **字段** | 与界面 Label 一致 | 若仅在某场景出现，可写为「实例名称（新建）」等 |
| **必填** | 是 / 否 / 条件必填 | 条件必填须写出触发条件 |
| **规则与说明** | 校验、长度、格式、默认值、只读等 | 如「2～50 个字符」 |
| **数据/选项来源** | 枚举、接口、本地 mock、计算字段 | 如「企业下拉：mock 企业列表」 |
| **备注** | 演示口径、与编辑差异、后续迭代 | 如「新建：…；编辑：…」「多实例后放宽过滤」 |

**新建与编辑规则不一致时**（在不拆条目的前提下）：统一写在 **备注** 列，使用「新建：…；编辑：…」或「仅新建」「编辑只读」等明确句式；必要时在 **字段** 列用场景后缀区分。

---

## 4. 与现有实现的关系

- 当前 `ModuleDocsPage` 为「分组 → 子模块标题 → `bullets: string[]`」。
- 实现阶段（另见 implementation plan）：在数据结构中为每个 `item` 增加可选的「字段标准」数据（例如 `fieldStandards?: FieldStandardRow[]`），渲染时在该 `item` 的 **标题下方、bullets 上方** 渲染表格；无数据时不展示表或展示占位句。
- 样式与现有控制台一致：沿用该页已有卡片、边框与字号层级；不引入第二套组件语言。

---

## 5. 非目标（本期不做）

- 不从该页反向生成表单校验代码或单一事实来源（SSOT）；文档为**人工维护**的说明，与代码可对齐但不必自动同步。
- 不在本期要求「全站所有页面一次填完」；允许按优先级逐页补全字段表。

---

## 6. 验收要点

- 读者打开任一已配置的模块条目，**最先**看到字段标准表（或明确无字段口径的说明），再看到原有说明 bullets。
- 五列含义在团队内一致；新增模块作者可按表填空，无需再 invent 结构。
